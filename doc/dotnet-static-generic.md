# The `dotnet:static-generic` Macro in `dotcl`

The `dotnet:static-generic` function allows the Lisp environment to call static `.NET` methods that require explicit generic type arguments. Because `.NET` reflection requires generic parameters to be bound (closed) via `MethodInfo.MakeGenericMethod` before invocation, this function handles the lookup, type-resolution, specialization, and execution.

## Syntax in Lisp

```lisp
(dotnet:static-generic type-name method-name type-arguments &rest method-arguments)
```

## Parameters

1. **`type-name`** *(String)*:
   The name of the `.NET` class containing the static method. This is resolved dynamically by `ResolveDotNetType` (which scans loaded assemblies). It can be a fully-qualified name or a bare name if the assembly is already loaded.
   * *Example:* `"System.Linq.Enumerable"` or `"Microsoft.Xna.Framework.Content.ContentManager"`

2. **`method-name`** *(String)*:
   The name of the generic static method to call.
   * *Example:* `"Where"` or `"Load"`

3. **`type-arguments`** *(List of Strings)*:
   A standard Lisp list containing the names of the types to bind to the method's generic parameters. These names are also resolved to `System.Type` instances dynamically.
   * *Example:* `'("System.Int32")` or `'("Microsoft.Xna.Framework.Graphics.Texture2D")`

4. **`method-arguments`** *(rest)*:
   The actual arguments to pass to the method. `dotcl` automatically handles converting Lisp types (integers, strings, floats, classes) to their `.NET` representations. Furthermore, it automatically marshals Lisp functions (`LispFunction` / lambda objects) into concrete C# delegate types (like `Func<T, bool>`) based on the method parameter signatures.

## Under the Hood Resolution

When `dotnet:static-generic` is called, the `dotcl` runtime:
1. Resolves the host type from `type-name`.
2. Searches for generic static method definitions (`IsGenericMethodDefinition == true`) that:
   * Match the given `method-name`.
   * Have the same number of generic parameters as the length of `type-arguments`.
   * Have the same number of parameters as the number of provided `method-arguments`.
3. Resolves the type names in `type-arguments` into `System.Type` objects.
4. Call `methodInfo.MakeGenericMethod(resolvedTypes)` to construct the specialized method definition.
5. Converts/coerces the Lisp arguments to match the parameter types of the specialized method and invokes it.

## Usage Example: If `ContentManager.Load<T>` Were Static

If `Load<T>` were a static method on `ContentManager` defined as `public static T Load<T>(ContentManager content, string assetName)`:

```lisp
(let* ((cont (content game))
       (logo (dotnet:static-generic 
               "Microsoft.Xna.Framework.Content.ContentManager" 
               "Load" 
               '("Microsoft.Xna.Framework.Graphics.Texture2D") 
               cont 
               "images/logo")))
  ;; logo is now a specialized Texture2D object
  (setf (logo game) logo))
```
