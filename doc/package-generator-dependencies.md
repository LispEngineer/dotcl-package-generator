# Package Generator Dependencies

* Copyright 2026 Douglas P. Fields, Jr.

This document outlines the dependencies that the Lisp C# package generator and the generated wrappers have on non-standard, custom code in this repository (excluding standard Common Lisp symbols and standard DotCL features).

## 1. Dependencies of the Package Generator Code (`assembly-package-generator.lisp`)

The generator depends on custom utility functions defined in other source files in the project:

*   **`utils:safe-read-form-from-file`** (defined in [utils.lisp](file:///home/dfields/src/cl/dotcl-dungeonslime/utils.lisp)): Used to read metadata lists from the temporary metadata files generated by the `AssemblyToLispy` tool.
*   **`utils:format-red`** (defined in [utils.lisp](file:///home/dfields/src/cl/dotcl-dungeonslime/utils.lisp)): Used for color-coded high-visibility terminal printing of warning and error messages.
*   **`monoutils:dotnet-p`** (defined in [monoutils.lisp](file:///home/dfields/src/cl/dotcl-dungeonslime/monoutils.lisp)): Used inside generator helper functions to determine types and generate type-check expressions.
*   **`monoutils:get-type`** (defined in [monoutils.lisp](file:///home/dfields/src/cl/dotcl-dungeonslime/monoutils.lisp)): Used in code-generation blocks to emit type retrievals.
*   **`uiop`** (ASDF/Common Lisp Standard Utility Library):
    *   `uiop:string-prefix-p`
    *   `uiop:read-file-string`
    *   `uiop:directory-files`

## 2. Dependencies of the Generated Packages (`cspackages/*.lisp`)

The generated C# package wrappers contain emitted Lisp code that relies on the following symbols at runtime:

*   **`monoutils:dotnet-p`** (defined in [monoutils.lisp](file:///home/dfields/src/cl/dotcl-dungeonslime/monoutils.lisp)): Used within the master overload `cond` dispatch structures to verify if an argument is a native .NET object (e.g., `(monoutils:dotnet-p value)`).
*   **`monoutils:get-type`** (defined in [monoutils.lisp](file:///home/dfields/src/cl/dotcl-dungeonslime/monoutils.lisp)): Evaluated at package load time to bind type constants (e.g., `(cl:defconstant <type> (monoutils:get-type "Microsoft.Xna.Framework.Vector2"))`).
*   **`utils:csharp-overload-not-found`** (defined in [utils.lisp](file:///home/dfields/src/cl/dotcl-dungeonslime/utils.lisp)): Raised as a condition type in the fallback branch of the `cond` block when no C# overload signature matches the supplied parameters.
